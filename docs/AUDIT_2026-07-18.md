# Correctness Audit — 2026-07-18

Multi-agent audit (121 agents, 8 dimensions: tax-calc, filings, stub-flow, pdf-docs,
dates-tz, w2-flow, ui-consistency, live-data). Every finding was adversarially
verified by two independent agents. Result: **55 confirmed, 1 plausible, 0 refuted**
(~40 distinct after merging duplicates), plus 3 security concerns from a
completeness critic.

Status legend: `FIXED 2026-07-18` = fixed same day on branch
`bugfix/nys45-q2-discrepancies`; `FIXED 2026-07-18 (remediation)` = fixed on
branch `bugfix/audit-remediation`; everything else is open.

## Drop-everything items

1. **W-2 generation completely broken** — `src/app/api/w2/calculate/route.ts:26`.
   Employee lookup used `.single()` but prod has two employee-role profiles (real +
   permanent test account from migration 0026), so every Preview W-2 click 500s.
   First W-2 legally due Jan 31, 2027. **FIXED 2026-07-18** (`.order('is_test').limit(1).maybeSingle()`).
2. **RLS bypass via `paystubs_with_tax_year` view** — the view ran with owner
   (postgres) privileges and `authenticated` had SELECT, so the employee login could
   read every stub row incl. zelle_transaction_id and employer taxes.
   **FIXED 2026-07-18** (migration 0035: `security_invoker = true`).
3. **MFA enforced for pages but not API routes** — `src/proxy.ts:51`. An aal1
   session could pull `/api/stubs/export` CSV and stub PDFs. **FIXED 2026-07-18**
   (API routes now 401 at aal1; `/api/auth/sign-out` exempt so a stuck user can escape).
4. **Reminder cron has never run successfully** — `CRON_SECRET` was never created in
   the Vercel project, so every scheduled call 401'd (route compared against literal
   `Bearer undefined`); no reminder email has ever been sent. Code hardened
   2026-07-18 (fail closed + distinct 500 when unset; wrong comment fixed).
   **OPEN: set CRON_SECRET in Vercel prod env and redeploy.**

## High severity

5. **Year-frozen YTD + tax rates on pay-date edit** — `src/app/(app)/stubs/new/page.tsx:78`
   (also `stubs/[id]/edit/page.tsx:30`). YTD wage-base context and tax_rates year are
   fetched for the suggested pay date, never re-derived when the admin edits it across
   a year boundary. Example: stub moved Dec 31 2026 → Jan 2 2027 saves FUTA $0.00
   instead of $1.19 and applies 2026 rates/accumulators to a 2027 stub. Verifiers:
   NYS-45/Schedule H recompute from gross and self-correct; exposure is per-stub
   ledger, 1040-ES sums, HYSA amounts ($1–25/boundary stub). Variant: YTD query has
   no upper year bound and no prior-stub predicate, so backfilling a missed week
   counts later-paid stubs as prior wages. **FIXED 2026-07-18 (remediation)**
   (generatePreview() re-derives YTD context + tax rates for the entered pay date;
   server queries bounded to the suggested/stored date).
6. **Reminder dismiss roll-forward writes wrong title year** — `src/components/reminders/RemindersView.tsx:53`.
   Successor year derived from due-date year + 1, but Schedule H / NYS-45 Q4 /
   1040-ES Q4 / W-2 titles carry the tax year (due year − 1). Dismissing
   "Schedule H 2025" creates "Schedule H 2027", skipping 2026; page then totals the
   wrong year's stubs and mark-filed records the wrong tax_year. Fix like sibling
   UpcomingDeadlines.tsx (increment the title's own matched year).
   **FIXED 2026-07-18 (remediation)** (shared rollForwardReminder() in
   lib/reminders.ts, used by both dismiss paths; unit-tested; no live rows damaged).
7. **W-2/W-3 shifted due date perpetuated** — `RemindersView.tsx:52`. Migration 0028
   stored shifted 2027-02-01 instead of statutory Jan 31; roll-forward keeps
   month-day, so 2028 shows Feb 1 when the real deadline is Mon Jan 31, 2028
   (SSA late penalty ~$60–130). Store statutory dates; shift at display time.
   **FIXED 2026-07-18 (remediation)** (migration 0036 reverts the live row to
   2027-01-31; RemindersView already shifts at display via shiftedDeadline()).
8. **Duplicate HYSA deposit on every note edit** — `src/components/stubs/StubDetail.tsx:148`.
   Edit-note dialog's Update unconditionally calls insertHysaDeposit() (no dedupe, no
   unique constraint), inflating the reserve balance by a full deposit per edit. Fix:
   insert only when not yet transferred + partial unique index on paystub_id.
   **FIXED 2026-07-18 (remediation)** (note edits no longer insert or re-stamp
   hysa_transferred_at; migration 0037 adds the partial unique index; no live dupes).
9. **Dashboard filing card tracks accruing quarter, not the one due** — `src/components/dashboard/AdminDashboard.tsx:164`.
   During the whole July window it shows "Q3 due Oct 31" while unfiled Q2 is due
   Jul 31; overdue/countdown badges unreachable. Show previous quarter until filed.
   **FIXED 2026-07-18 (remediation)** (previousQuarter() helper; card shows the
   prior quarter until it's filed or marked N/A, then the accruing one).
10. **Re-rendered stub PDFs include future stubs in line-item YTD** — `src/app/api/pdf/stub/route.ts:82`
    (+ email route + `stubs/[id]/page.tsx:93`). Bonus/reimbursement YTD sums include
    stubs paid after the one rendered; NY §195(3) statement internally inconsistent
    on re-download. Filter to prior + current stubs in all three call sites.
    **FIXED 2026-07-18 (remediation)** (all three call sites now sum line items
    over prior + current stub ids only).
